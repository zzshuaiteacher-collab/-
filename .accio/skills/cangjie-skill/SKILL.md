# 仓颉 (Cangjie) v2.0 · 高仿生写作引擎

> 「理以察势，文以载道，情以动人，神以进化。」

## 核心理念
仓颉不是「模仿文字」，是 **「穿上灵魂」**。
一个好的仿生写作包是一套可运行的「写作操作系统」：
- 他们用什么 **心智模型** 看世界？（理）
- 他们用什么 **句式和节奏** 表达？（文）
- 他们在文字中注入什么 **情感基调和共情模式**？（情）
- 他们的风格如何在用户反馈中 **持续进化**？（神）

**关键区分**：仓颉捕捉的不仅是 WHAT they write 和 HOW they write，更是 **WHY they feel that way when writing**。

## 执行流程 (The Cangjie Swarm v2.0)

### Phase 0: 目标锚定与入口分流

| 用户输入 | 路径 | 示例 |
|---------|------|------|
| 明确的团队/公众号名 | **直接路径** → Phase 0A | 「蒸馏极客公园的风格」 |
| 模糊的写作需求 | **诊断路径** → Phase 0B | 「我想写出晚点那种冷峻感」 |

---

### Phase 0A: 需求澄清（直接路径）
收到明确名字后，确认：
1. **目标确认**：这个团队/公众号是谁？领域是什么？
2. **聚焦方向**（可选）：全面画像 vs 聚焦某个维度（如只要情感基调）？
3. **写作用途**：模仿写作？风格学习？内容生产自动化？
4. **新建 or 更新**：检查 `.accio/skills/` 是否已有该风格包。

用户说「就蒸馏XX」→ 默认全面画像，直接推进到 Phase 0.5。

---

### Phase 0B: 需求诊断（模糊路径）
用户不知道该学习谁，只有模糊需求。仓颉从需求反推最适合的风格模板。

#### Step 1: 风格需求定位
通过 1-2 个追问，定位核心诉求：

| 需求维度 | 典型表达 | 推荐方向 |
|---------|---------|---------|
| 深度商业分析 | 「想写出有洞察力的商业长文」 | 晚点、42章经 |
| 科技快报 | 「想第一时间解读AI突破」 | 新智元、量子位 |
| 产业深度 | 「想拆解一个行业的供应链逻辑」 | 远川研究所 |
| 创新叙事 | 「想写出酷炫的创新故事」 | 极客公园 |
| 情感共鸣 | 「想让读者读完有感触」 | 人物、GQ报道 |
| 讽刺犀利 | 「想写得有态度有锋芒」 | 虎嗅、差评 |

#### Step 2: 候选推荐
每个候选的展示格式：
```
### 候选1: [团队名] ⚡已有风格包 / 🆕需要蒸馏
**核心风格**：[一句话描述其写作灵魂]
**情感基调**：[冷峻/热情/讽刺/温情/焦虑...]
**为什么适合你**：[直接对应用户需求]
**局限**：[这种风格的盲区]
```

#### Step 3: 用户选择
- 选了已有风格包 → 直接激活
- 选了新蒸馏候选 → 进入 Phase 0A → Phase 0.5

### Phase 1: 9路并行地毯式调研 (9-Agent Swarm)

同时启动 9 个专业 Sub-Agents。每个 Agent 的调研结果**必须写入对应的 .md 文件**。禁止在没有文件留痕的情况下直接总结。

| # | Agent 名称 | 搜索目标 | 输出文件 |
|---|-----------|---------|---------|
| 1 | **著作 Agent** | 书籍、长篇Newsletter、系统性论述 | `01-writings.md` |
| 2 | **对话 Agent** | 播客、视频访谈、AMA、即兴表达 | `02-conversations.md` |
| 3 | **表达 Agent** | 社交媒体碎片、微博、Twitter | `03-expression.md` |
| 4 | **他者 Agent** | 行业评价、争议、同行对比 | `04-external.md` |
| 5 | **决策 Agent** | 选题偏好、资源投入逻辑、报道频率 | `05-decisions.md` |
| 6 | **演化 Agent** | 风格随时间轴的变迁、转型节点 | `06-evolution.md` |
| 7 | **微信白名单 Agent** | 公众号历史文章地毯式搜索、爆款采样 | `07-wechat.md` |
| 8 | **排版美学 Agent** | 加粗逻辑、标点密度、Emoji、配图调性、标题格式 | `08-layout.md` |
| 9 | **情感基调 Agent** ⭐新增 | 情感词频、情绪弧线、共情模式、情感禁区 | `09-emotion.md` |

#### Agent 9: 情感基调 Agent 详细指令
该 Agent 必须完成以下任务：
1. **情感词频扫描**：统计语料中正面词/负面词/中性词的比例。
2. **情绪弧线识别**：分析文章内情绪的起承转合（如：开头克制→中段紧张→结尾释放 OR 全程高能 OR 全程冷峻）。
3. **共情模式标注**：
   - **俯视型**：「我来告诉你真相」（如：晚点、远川）
   - **平视型**：「我们一起来看看」（如：极客公园、42章经）
   - **仰视型**：「这太牛了！」（如：新智元、量子位）
4. **情感禁区标注**：该团队绝对不会使用的情感表达（如：晚点绝不煽情、远川绝不自嗨）。
5. **温度计评分**：对该团队的整体情感温度打分（0=极度冷峻，100=极度热情）。

#### 信息源优先级
| 优先级 | 渠道 | 说明 |
|--------|------|------|
| ★★★ | 微信公众号原文 | 仓颉的一级水源，白名单最高权重 |
| ★★★ | 官方网站长文 | 通常是最完整的版本 |
| ★★☆ | 播客/视频实录 | 即兴表达中暴露真实风格 |
| ★★☆ | 36氪/晚点/极客公园等权威转载 | 二手但可信 |
| ★☆☆ | 社交媒体碎片 | 辅助验证，不作为主力 |
| ☆☆☆ | 知乎/百度百科 | 仅在其他渠道不足时作为补充 |

### Phase 2: 四重结构化提炼 (Quadruple Extraction)

9 个 Agent 的素材汇总后，执行四重提炼。

#### 2A: 理之提炼 (Logic Extraction)
执行「三重验证」：
1. **跨域复现**：在不同类型的文章中反复出现其分析逻辑。
2. **生成力**：可以用此模型推断该团队对新热点的态度。
3. **排他性**：具有该团队独特的智力标签。

**输出**：3-7 个心智模型，每个含证据、应用场景、局限。

#### 2B: 文之提炼 (Expression Extraction)
**定性描述**（写入 SKILL.md）：
- 高频术语、语气助词、标志性句式。
- 叙事模式（结论先行 / 剥洋葱 / 商业戏剧）。
- 排版习惯（加粗规则、标题格式、Emoji 使用频率）。

**定量参数**（写入 `Writing_Pattern.json`）：
```json
{
  "avg_sentence_length": 28,
  "paragraph_avg_lines": 4,
  "question_ratio": 0.12,
  "exclamation_ratio": 0.02,
  "bold_frequency": "every_3_paragraphs",
  "function_words": {
    "其实": "high",
    "本质上": "high",
    "有趣的是": "medium",
    "毕竟": "low"
  },
  "punctuation_density": {
    "comma_per_100chars": 6.2,
    "period_per_100chars": 3.1,
    "exclamation_per_100chars": 0.3,
    "question_per_100chars": 1.1
  },
  "quote_style": "「直角引号」",
  "title_format": "01 / 02 / 03 加粗数字"
}
```

#### 2C: 情之提炼 (Emotion Extraction) ⭐新增
从 Agent 9 的调研结果中提炼以下四个维度：

| 维度 | 说明 | 示例 |
|------|------|------|
| **Valence (效价)** | 正面/负面/中性的基准比例 | 晚点: 正20% 负30% 中50% |
| **Arousal (唤醒度)** | 高能/平静/压抑的基准比例 | 新智元: 高能70% 平静20% 压抑10% |
| **Empathy (共情模式)** | 俯视/平视/仰视 | 极客公园: 平视为主，偶尔仰视 |
| **Tension Arc (情感弧线)** | 文章内情绪的起承转合规律 | 42章经: 冷启动→逻辑递进→洞察高潮→冷收尾 |

**额外输出**：
- **情感温度计**：0-100 分，标注该团队的整体情感温度。
- **情感禁区清单**：列出 3-5 条绝对不能出现的情感表达。

#### 2D: 黄金语料锚定 (Golden Corpus Anchoring) ⭐新增
从采集的语料中精选 **3-5 段最具代表性的原文段落**（每段 200-500 字），写入 `Golden_Corpus.md`。

每段标注：
```markdown
### 语料 1: [文章标题]
**来源**: [URL]
**情感基调**: [冷峻/热情/讽刺...]
**叙事模式**: [结论先行/剥洋葱/商业戏剧]
**为什么选这段**: [它最能代表该团队的写作灵魂]

> [原文段落，200-500字]
```

**用途**：在 Phase 3 仿生写作时，将黄金语料作为 few-shot 示例直接注入 Prompt，确保风格锚定。

### Phase 3: 仿生构建与补丁对齐 (Construction)

1. 读取 `references/skill-template.md`，按照模板生成 `SKILL.md`。
2. 将 `Writing_Pattern.json` 的量化参数嵌入写作指令。
3. 将 `Golden_Corpus.md` 的原文段落作为 few-shot 示例嵌入。
4. **补丁优先级**：若存在 `Correction.md`，强制将其规则置于最高权重。

#### 补丁加载顺序（权重从高到低）：
```
Correction.md (用户纠错补丁) → 最高权重
Golden_Corpus.md (黄金语料锚点) → 第二权重
Writing_Pattern.json (量化参数) → 第三权重
SKILL.md (基础框架) → 基础权重
```

---

### Phase 4: 三级验证与反馈进化 (Validation & Evolution)

#### 4A: 仿真写作测试
模拟该团队撰写一段 **1000 字以上的深度分析**，覆盖其核心选题领域。

#### 4B: 风格审计 Agent ⭐新增
启动一个独立的 Sub-Agent，执行自动化风格匹配评估：

| 检测维度 | 方法 | 通过标准 |
|---------|------|---------|
| **词汇匹配** | 对比生成文本与黄金语料的高频词重合度 | ≥ 80% |
| **句长分布** | 对比平均句长与 Writing_Pattern.json | 偏差 ≤ 15% |
| **情感温度** | 对比生成文本与情感温度计评分 | 偏差 ≤ 10分 |
| **叙事模式** | 验证文章结构是否符合 SKILL.md 中的叙事模式 | 一致 |

**输出**：仿生指数（0-100）。
- **≥ 90**：通过，交付。
- **85-89**：微调后交付（自动调整 Writing_Pattern 参数）。
- **< 85**：打回 Phase 2，重新提炼。

#### 4C: 用户反馈进化
接收用户的自然语言反馈，自动分类并写入 `Correction.md`：

| 反馈类型 | 分类标签 | 示例 |
|---------|---------|------|
| 逻辑层纠错 | `[C-理]` | 「晚点不会用这个论证结构」 |
| 表达层纠错 | `[C-文]` | 「42章经从不用感叹号」 |
| 情感层纠错 | `[C-情]` | 「极客公园在这里应该更兴奋」 |
| 排版层纠错 | `[C-排]` | 「远川的加粗频率应该更低」 |

**Correction.md 格式**：
```markdown
# 进化纠错记录

## [C-情] 2026-04-09
- 用户反馈：「这段写得太冷了，极客公园在谈到创新者时应该有一种发自内心的赞赏」
- 修正规则：当主题涉及创新者/创始人时，情感温度 +20，共情模式切换为「平视偏仰视」

## [C-文] 2026-04-09
- 用户反馈：「42章经不会用这么多感叹号」
- 修正规则：exclamation_per_100chars 上限从 0.3 调整为 0.05
```

## 品味守则

| 原则 | 一句话 |
|------|--------|
| 长文 > 金句 | 3000字深度文比50条推文更能揭示风格全貌 |
| 争议 > 共识 | 最被争议的文章最能暴露独特性 |
| 情感 > 修辞 | 读者记住的不是句式，是感受 |
| 原文 > 描述 | 10段黄金语料胜过100条规则描述 |
| 矛盾 > 一致 | 风格中的内在张力是人格的核心特征 |

## 仓颉禁令
- 严禁在调研不足时凭空想象风格。
- 严禁将公众号移入黑名单（它是仓颉的一级水源）。
- 严禁简化排版美学的提炼（视觉也是文笔的一部分）。
- 严禁忽略情感层（没有情感的文字是死的）。
- 严禁用定性描述替代定量参数（「短句风格」不够，必须给出平均句长）。
- 严禁跳过黄金语料锚定（没有 few-shot 的仿生是盲人摸象）。
- 严禁在仿生指数 < 85 时交付成品。
