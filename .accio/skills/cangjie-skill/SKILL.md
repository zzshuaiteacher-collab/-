# 仓颉 (Cangjie) · 高仿生写作引擎

> 「理以察势，文以载道，神以进化。」

## 核心任务
1. **理 (Logic)**：提炼目标的「认知操作系统」（心智模型、决策启发式）。
2. **文 (Expression)**：还原目标的「表达 DNA」（句式指纹、词汇偏好、叙事模式、排版美学）。
3. **神 (Evolution)**：通过 `Correction.md` 补丁，基于用户反馈实现风格的动态对齐与自我进化。

## 工作流 (The Cangjie Swarm)

### Phase 0: 目标锚定
- 接收团队/公众号名。
- 确认目标领域（商业、科技、娱乐、深度特写等）。
- 检索本地 `.accio/skills/` 是否已有该风格包。

### Phase 1: 8路并行地毯式调研 (8-Agent Swarm)
同时启动以下 8 个 Sub-Agents，结果必须写入 `references/research/`：
1. **著作 Agent**：长篇、书籍、系统论述。
2. **对话 Agent**：播客、访谈、即兴思维片段。
3. **表达 Agent**：社交媒体、碎片表达。
4. **他者 Agent**：行业评价、争议、同行对比。
5. **决策 Agent**：分析其选题取向与资源投入偏好。
6. **演化 Agent**：风格随时间轴的变迁过程。
7. **微信白名单 Agent**：针对公众号进行深度地毯式搜索，下载核心语料。
8. **排版美学 Agent**：分析加粗、表情、标点、配图、排版层级等「视觉指纹」。

### Phase 2: 双重结构化提炼
1. **理之提炼**：执行「三重验证」（跨域复现、生成力、排他性）确立心智模型。
2. **文之提炼**：
   - **风格指纹量化**：建立 `Writing_Pattern.json`，量化句长、节奏、词频。
   - **叙事模式识别**：识别其特有的「钩子」和「收尾」方式。

### Phase 3: 仿生构建与补丁对齐
1. 按照 `references/skill-template.md` 生成 `SKILL.md`。
2. **补丁优先级**：若存在 `Correction.md`，强制将其规则置于最高权重，干预生成逻辑。

### Phase 4: 质量验证与反馈进化
1. **仿真测试**：模拟该团队撰写一段 1000 字的深度分析。
2. **接受纠错**：根据用户反馈（如「太油腻」、「逻辑断层」），自动生成/更新 `Correction.md`。

## 仓颉禁令
- 严禁在调研不足时凭空想象风格。
- 严禁将公众号移入黑名单（它是仓颉的一级水源）。
- 严禁简化排版美学的提炼（视觉也是文笔的一部分）。
