# 仓颉提炼方法论：理与文的耦合

## 一、 理之提炼：心智模型的三重验证 (借鉴女娲)
每一个被收录的「模型」必须满足：
1. **跨域复现**：在不同类型的文章中反复出现其分析逻辑。
2. **生成力**：可以用此模型推断该团队对新热点的态度。
3. **排他性**：具有该团队独特的智力标签（如：远川的「底座」、42章经的「元问题」）。

## 二、 文之提炼：表达 DNA 的量化 (借鉴同事)
通过对海量语料（含微信白名单）的扫描，提取以下指纹：

### 1. 词汇指纹
- **高频术语**：该行业中该团队偏好的特定词汇。
- **语气助词**：结尾习惯（嘛、呢、吧）或转折习惯（其实、毕竟、有趣的是）。

### 2. 节奏指纹
- **平均句长**：是海明威式的短句，还是普鲁斯特式的长句？
- **段落密度**：是否频繁换行？
- **反问频率**：是否喜欢用设问句驱动叙事？

### 3. 叙事模式
- **结论先行型**：开篇炸场，中间堆料。
- **剥洋葱型**：从细节切入，层层递进。
- **商业戏剧型**：强人设、强冲突、强对话。

## 三、 神之进化：Correction 机制
当用户提出「不像」或「不对」时，执行以下操作：
1. **识别冲突**：定位是逻辑错误（理）还是风格偏差（文）。
2. **生成补丁**：在 `Correction.md` 中写入一条禁止或必须的指令。
3. **权重复写**：在下一次 `write` 或 `edit` 前，强制将补丁内容注入 Prompt 的最顶层。

## 四、 微信白名单检索策略
1. **采样策略**：优先采样过去 3 个月的 Top 5 爆款文章。
2. **视觉抓取**：记录文中加粗（Bold）、倾斜（Italic）及 Emoji 的具体坐标和频率。
